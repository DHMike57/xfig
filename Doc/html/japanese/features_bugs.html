新しい機能と修正された問題点

xfig 3.2.1 の新しい機能

  • ライブラリ機能 ― Fig オブジェクトのライブラリをロードし、 そのライブラリから選択したオブジェクトをキャンバスに入力することができます。 詳細は Libraries を参照して下さい。 また、ライブラリの例については Examples/Libraries ディレクトリを 参照して下さい。 From Stephane Mancini (mancini@elecsrv.enst.fr) with modifications by Brian V. Smith

  • Fig オブジェクトをカット・バッファから ペーストする際に、 そのオブジェクトをマウス・カーソルの下 (Paste ボタンが押された場合にはその近く) に表示し、その状態で位置を移動させた後で マウスボタン1をクリックすることによって指定した場所に置くようにしました。

  • UNIT ボックスに スケール (例えば 1cm=3km) を表示するようにしました。 また、この変更に伴って、UNIX ボックスを広げるために 横のルーラをやや広くしました。

  • xfig のチュートリアル (PDF)、 Doc/xfig-howto.pdf。
    written by Peter Hiscocks (phiscock@ee.ryerson.ca)

  • Electrical、及び Logic ライブラリに含まれている ライブラリ・オブジェクトは、 Peter Hiscocks (phiscock@ee.ryerson.ca) によって作成されました。

  • PICTURE オブジェクトのファイル名を、 それがその Fig ファイルのサブ・ディレクトリに置かれている場合には相対パス、 さもなくば絶対パスとしてセーブするようにしました。 From T. Sato

  • File パネルでファイル名をクリックした際に、 その Fig ファイルの図のプレビューを表示するようにしました。

  • Fig ファイルをマージした際には、 さらに他のファイルをマージすることができるように File パネルを開いたままにするようにしました。

  • オブジェクトのライブラリを、 Fig.library_dir リソース、もしくは -library_dir オプション によって指定できるようにしました。 ディレクトリを指定した場合には、 ライブラリとなるサブ・ディレクトリと *.fig ファイルが そのディレクトリから探されます。

  • フリーハンドの折れ線、多角形、スプラインの入力 ― POLYLINE、POLYGON、SPLINE などの描画モードを選択した後、 入力を開始するためにマウスボタン2をクリックしてマウスを動かすと、 線に点が追加されていきます。 詳細は POLYLINES を参照して下さい。
    [注意] これに伴って描画モードの挙動が変更されています。 単一の点はマウスボタン3によって入力するようになりました。

  • Print パネルが開かれている時でも Print ボタンをアクティブのままとするようにしました。 これは、そのパネルが他のウィンドウに隠れている場合に、 それを容易にウィンドウの重なりのトップに動かせるようにします。

  • Export パネルが開かれている時に コマンド・パネルの File ボタンがクリックされた場合には Export パネルを閉じて File パネルを開くようにしました。 その逆も同様であり、File パネルが開かれている時に Export ボタンがクリックされた場合には File パネルを閉じて Export パネルを開きます。

  • 編集中の図にファイルをマージする際に、 それをあらかじめ1個の複合オブジェクトにまとめるようにしました。

  • オブジェクトの描画に際して一時的に用いる配列のメモリ割り当ての改良 ― 若干早くなるかも知れない。 From T. Sato

  • 取り込まれた PICTURE オブジェクト (EPS、GIF など) のファイル名に 絶対パスを用いるようにしました。

  • PICTURE オブジェクトの Edit パネル において、画面上の指定した領域の取り込みに加え、 ウィンドウや画面全体の取り込みを行なえるようにしました。 マウスボタン1は指定したウィンドウ全体を取り込み、 マウスボタン2は画面上の指定した領域を取り込むようになっています。

  • Fig*browse_panel*mask*string に文字を追加し、 *.tif、及び *.sld ファイルを含むようにしました。

  • xfig がファイル名を指定せずに起動した際の 初期状態でのファイル名を unnamed.fig ではなく空としました。 これは、セーブに際して File パネルを開くようにします。

  • バルーン・メッセージを表示するまでの遅延時間を、 -balloon_delay オプション、 もしくは Fig.balloon_delay リソースによって ミリ秒単位で指定できるようにしました。

  • File パネル、Export パネル、もしくは Library パネルを拡大した際に、 より多くのファイルを表示できるように File リストのみを拡大するようにしました。

  • インジケータ・ボタン (例えば LINE WIDTH) のポップアップ・パネルなどで、 マウス操作によって数値を増加/減少させられるように、 新たに ``spinner'' ウィジェットを使うようにしました。

  • 矢印の入力に際しての矢印の太さ、幅、及び長さを、インジケータ・パネルの ARROW SIZE ボタンによって 設定できるようにしました。 UPDATE は、インジケータ・パネルとオブジェクトの間で矢印の大きさもコピーします。

    矢印の幅や長さの単位は矢印の線の太さと同じもの (これは線の太さの単位と同一) に変更されており、 xfig の以前のバージョンで用いられていた1/1200インチではなくなりました。

  • Edit パネルの Color メニューを、 多くの色をよりうまく取り扱えるように変更しました。

  • コンパイルに際して Imakefile で USESMALLICONS を指定することにより、 描画/編集モード・パネルに小さいアイコンを使用できるようにしました。 詳細については Imakefile を参照して下さい。 また、Fig.ad も参照 (icons を探す) して下さい。 From Andrew Morgan (morgan@physics.ucla.edu)

  • オブジェクトの入力に際して depth を 0 よりも大きい値に設定しておくことを憶えておかなくても、 新しいオブジェクトを容易にその上に置くことができるように、 depth の初期値を 100 に変更しました。

  • GRID MODE、 及び POINT POSITION として、1/8インチ (もしくは 2mm) を指定できるようにしました。

  • 折れ線、スプライン、及び円弧のスケーリングに際して、 矢印もスケーリングするようにしました。

  • オブジェクトの移動やコピー、オブジェクトの点の移動や追加に際して、 図が見やすくなるように 他のオブジェクトのコーナー・マーカーを一時的に消すようにしました。 それらは、操作が完了した時点で再び表示されます。

  • メニュー・ボタンに、それがメニュー・ボタンであることを示すために 矢印のアイコンを付けるようにしました。

  • Edit パネル での座標の入力に際して、 数値の後に ``c'' を付けることによって cm 単位、 ``i'' を付けることによって inch 単位で入力することができる (例えば、3c は 3cm) ようにしました。 From Rick Richardson (rick@dgii.com)

  • PICTURE ファイル (EPS、GIF、その他) の読み込みに際して、 そのフォーマットの判別をより賢く、かつ素早く行なうようにしました。

  • Integral area zoom の指定 ― これが設定されている場合には、 キャンバス上で領域を指定して拡大を行なう場合の拡大率が整数 (例えば、2、3、4 など) に制限されます。

  • Sun SPARCstation において、 Compose キャラクタ (a-umlaut など) の入力に際して Compose キーの LED を点灯させられるようにしました。 README ファイルの COMP_LED を参照して下さい。

  • fig2dev の新しい Tk (tck/tk) ドライバ ― Tk のキャンバスとキャンバス・オブジェクトを生成します。 From Mike Markowski (mm@udel.edu)

  • 矢印の大きさを、絶対的な大きさとするか、 現在の線の太さに対する比率とするかを選択できるようにしました。
